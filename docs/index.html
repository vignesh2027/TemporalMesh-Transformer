TemporalMesh Transformer

A research transformer with dynamic graph attention, learned temporal decay, and per-token adaptive depth routing — three orthogonal innovations over the standard transformer baseline.

Motivation

Why TemporalMesh?

Standard transformers make three implicit assumptions that TMT relaxes: all tokens attend to all other tokens (O(S²) cost), all positions contribute equally regardless of distance, and every token passes through every layer. TMT challenges all three.

Feature Standard Transformer TemporalMesh Transformer
Attention complexity O(S²) — quadratic O(S·k) — sparse mesh
Graph topology Fixed all-to-all Dynamic kNN, rebuilt every layer
Position encoding Fixed RoPE / sinusoidal RoPE + learned decay scalars
Compute per token Always N layers Adaptive 1–N (exit gate)
FFN structure Single MLP Dual-stream (syntax + semantic)
Cross-sequence memory None Persistent anchors (EMA)
Training objective CE only CE + exit gate auxiliary
The Three Innovations

Core Technical Contributions

1 — Dynamic Mesh Attention

At every forward pass, each token is connected to its k nearest neighbours by cosine similarity of the current representations. Multi-head attention is constrained to these edges, giving O(S·k) cost instead of O(S²). The graph is rebuilt after every layer, so the communication structure adapts as representations evolve.

sim_ij = normalize(x_i) · normalize(x_j) E = { (i,j) : j ∈ top-k(sim_i) } scores = QK^T / sqrt(d_head) if (i,j) ∈ E else -inf attn = softmax(scores) out = attn · V

2 — Temporal Decay Position Encoding

Standard RoPE is augmented with per-token, per-dimension learned sigmoid decay scalars. Tokens at larger positions are attenuated before entering attention, giving the model an inductive bias toward recency. The same scalars further modulate per-head attention weights inside MeshAttention.

t_s = s / (S - 1) # normalised position in [0,1] decay_sd = sigmoid(-t_s * w_d) # w_d learnable (D,) x_s = RoPE(x_s) * decay_sd # attenuated embedding # Inside MeshAttention: head_decay = sigmoid(w_decay_h * mean(decay_s)) attn = softmax(scores) * head_decay

3 — Adaptive Depth Exit Gate

After each layer's attention + FFN sub-blocks, a lightweight Linear→Sigmoid gate scores each token's confidence. Tokens above the threshold are frozen and skip all remaining layers, reducing average compute on easy inputs. An auxiliary loss penalises uncertain (≈0.5) confidence scores.

confidence_s = sigmoid(W_gate · x_s + b_gate) exit_s = confidence_s > threshold OR prev_exit_s x_s = x_frozen_s if exit_s else updated_x_s # Auxiliary loss (encourages decisive 0-or-1 gates): gate_loss = -mean(|confidence - 0.5|) total = CE(logits, targets) + 0.1 * gate_loss
Architecture

Full Model Structure

TMTModel ├── TokenEmbedding nn.Embedding(V, D) × sqrt(D) ├── TemporalPositionEncoder RoPE + sigmoid decay scalars w_decay:(D,) ├── MeshBuilder kNN graph — O(S·k) edges per batch item │ └── TMTLayer × N ├── LayerNorm ├── MeshAttention Q/K/V projections + sparse graph masking │ └── w_decay:(H,) per-head temporal decay scalars ├── ── residual ── ├── LayerNorm ├── DualStreamFFN │ ├── syntax_up Linear(D→s) → GELU → syntax_down Linear(s→D) │ ├── semantic_up Linear(D→s) → GELU → semantic_down Linear(s→D) │ └── gate sigmoid(Linear(D→D)) weighted fusion ├── ── residual ── ├── ExitGate Linear(D→1) → sigmoid → threshold mask │ └── aux loss -mean(|conf - 0.5|) ├── LayerNorm ├── MemoryAnchorCross cross-attn(tokens → anchors) EMA update ├── ── residual ── └── freeze exited tokens x[exit_mask] = x_frozen[exit_mask] MeshBuilder(updated x) → rebuilt graph for next layer Final: LayerNorm → Linear(D, V, bias=False) [weight-tied to embedding]

Getting Started

Installation

1

Clone the repository

git clone https://github.com/vignesh2027/TemporalMesh-Transformer.git
cd TemporalMesh-Transformer
2

Install in editable mode (installs all dependencies)

pip install -e .
3

Run the test suite to verify everything works

pytest tests/ -v
4

Optional — install dev extras (ruff, pyright, bandit)

pip install ruff pyright bandit pytest-cov
Usage

Code Examples

Forward pass

import torch
from tmt.model.config import TMTConfig
from tmt.model.model  import TMTModel

cfg = TMTConfig(vocab_size=32000, d_model=512, n_heads=8,
                n_layers=12, graph_k=8, memory_anchors=16)
model = TMTModel(cfg)
model.eval()

ids = torch.randint(0, cfg.vocab_size, (1, 64))
with torch.no_grad():
    out = model(ids)

print(out.logits.shape)        # torch.Size([1, 64, 32000])
print(out.decay_scalars.shape) # torch.Size([1, 64, 512])
print(len(out.exit_masks))      # 12 — one per layer

Inspect exit rates per layer

for i, mask in enumerate(out.exit_masks):
    rate = mask.float().mean().item()
    print(f"Layer {i+1:2d}: {rate*100:.1f}% tokens exited")

Inspect the dynamic graph

edge_index, edge_weight = out.graph_edges
print(f"Edges: {edge_index.shape}")   # (2, E)
print(f"Weight range: [{edge_weight.min():.3f}, {edge_weight.max():.3f}]")

One training step

from torch.optim import AdamW
from tmt.training.loss      import compute_loss
from tmt.training.scheduler import cosine_warmup_scheduler

model.train()
opt  = AdamW(model.parameters(), lr=3e-4, weight_decay=0.1)
sched = cosine_warmup_scheduler(opt, warmup_steps=500, total_steps=10000)

ids     = torch.randint(0, cfg.vocab_size, (4, 257))
x, tgt  = ids[:, :-1], ids[:, 1:]
out     = model(x)
loss, ce, gate = compute_loss(out.logits, tgt, out.confidences)

opt.zero_grad(); loss.backward()
torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
opt.step(); sched.step()
print(f"loss={loss.item():.4f} ce={ce.item():.4f} gate={gate.item():.4f}")

API Reference

TMTOutput Dataclass

Every call to TMTModel.forward() returns a TMTOutput dataclass with the following fields:

Field Type Shape Description
logits Tensor (B, S, V) Next-token log-probabilities (unnormalised)
exit_masks List[Tensor] N × (B, S) bool Per-layer boolean exit mask — True where token exited
confidences List[Tensor] N × (B, S) float Sigmoid gate confidence scores ∈ (0, 1) per layer
graph_edges Tuple[Tensor, Tensor] (2,E) + (E,) Final layer edge index (COO) and cosine similarity weights
memory_state Tensor (M, D) Final memory anchor representations (detached)
decay_scalars Tensor (B, S, D) Temporal decay weights from TemporalPositionEncoder ∈ (0, 1)
Training

Training Output Explained

The trainer prints one line every 10 steps. Here is what each field means:

FieldMeaningHealthy range
lossTotal combined loss (CE + 0.1×gate)Decreasing over time
ceCross-entropy next-token predictionStarts at ~ln(V), decreases
gateExit gate auxiliary (negative, toward -0.5)−0.5 to 0.0
exitFraction of tokens that exited in final layer0–1; higher = more efficient
lrCurrent learning rate from cosine schedulerWarms up then decays
